# HJM 项目说明

## 项目概述

HJM 是一个使用中文字符编码的以太坊风格区块链实现。它将传统区块链中的十六进制地址和哈希值替换为"哈基米莫南北绿豆阿西呀库"等中文字符，创造了一个独特且有趣的区块链系统。

## 核心特性

### 1. 哈基米编码系统
- 使用27个不重复中文字符作为编码字符集
- 实现27进制 tryte 编码（每字节固定2个字符）
- 支持字节数据与中文字符串的双向转换

### 2. 完整的加密系统
- ECDSA签名算法（secp256k1曲线）
- Troika 哈希算法
- 与比特币/以太坊相同的签名标准

### 3. 区块链核心功能
- 完整的区块结构和链式存储
- 工作量证明（PoW）共识机制
- 交易创建、签名和验证
- 余额追踪和状态管理

### 4. 钱包系统
- 密钥对生成和管理
- 中文字符地址生成
- 交易创建和签名
- 私钥导入导出

## 技术实现

### 编码算法
```
字节数据 → 27进制拆分（每字节2字符）→ 中文字符串
```

### 地址生成
```
私钥 → 公钥 → Troika → 取后20字节 → 哈基米编码 → 地址
```

### 交易流程
```
创建交易 → 计算哈希 → 签名 → 添加到待处理池 → 挖矿打包 → 上链确认
```

## 文件结构

- `encoding.js`: 核心编码系统，实现字节与中文字符的转换
- `crypto.js`: 加密功能，包括签名、验证、哈希
- `transaction.js`: 交易数据结构和签名逻辑
- `block.js`: 区块结构和工作量证明
- `blockchain.js`: 区块链主逻辑，管理链和交易池
- `wallet.js`: 钱包功能，管理密钥和创建交易

## 使用场景

1. **教育演示**: 理解区块链原理的有趣方式
2. **概念验证**: 展示非传统编码在区块链中的应用
3. **创意项目**: 中文化的区块链实现

## 性能说明

- 编码/解码: 毫秒级
- 交易签名: 毫秒级
- 挖矿速度: 取决于难度设置
  - 难度1: 平均几秒到几十秒
  - 难度2: 平均几分钟
  - 难度3+: 可能需要很长时间

## 限制和注意事项

1. **仅供学习**: 这是一个教育项目，不适合生产环境
2. **挖矿性能**: 单线程 JavaScript 实现的挖矿效率较低
3. **网络功能**: 当前版本不包含P2P网络功能
4. **持久化**: 不包含数据库存储功能
